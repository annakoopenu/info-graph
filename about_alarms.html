About The Alarms Analysis

A short explanation of how the Haifa alarms dashboard was created.

What I Did

I took the raw alarms dataset from alarms.json and built a small Python analysis flow around it in alarms.py. The first step was to parse the file safely, including handling the malformed JSON shape that appeared in the source file.

Then I filtered the data to alarms related to Haifa by matching city names that contain Haifa or חיפה. From there I created a sub-dataset focused only on the relevant rows.

How The Analysis Was Built

  • Convert Unix timestamps into Israel local time using Asia/Jerusalem.
  • Extract useful fields such as date, hour, weekday, and matched Haifa areas.
  • Create a filtered CSV for the Haifa subset.
  • Explode multi-area alarms so one event can be analyzed under each matching Haifa area.
  • Generate visual summaries by hour, by date and hour, and by daily timeline.

Interactive Page

Finally, I generated haifa_alarms.html, a standalone HTML page that lets the user choose a Haifa area from a menu. By default it shows all areas together, and when an area is selected it redraws the graph to show the hourly alarm distribution for that area only.

The page also shows the date interval for the currently displayed data, so it is clear what period the graph refers to.
